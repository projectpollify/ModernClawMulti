# Suggestion System Spec

## Purpose

The suggestion system is the first differentiated product layer for ModernClaw.

Instead of silently rewriting memory files, the app should surface explicit, reviewable suggestions that help the user improve their AI brain over time.

## Core Idea

The system proposes updates in four broad categories:

1. profile suggestions
2. memory suggestions
3. knowledge suggestions
4. behavior suggestions

The user reviews and decides whether to accept, defer, or dismiss each one.

## Why It Matters

This creates a bridge between:

- blank memory files
- guided setup
- future cowork/curator agents

It allows the app to become progressively personalized without blocking first use.

## Suggestion Types

### Profile

Examples:

- add profession/context to `USER.md`
- add communication preferences
- add tools, devices, or work style

### Memory

Examples:

- add a current project to `MEMORY.md`
- store a durable preference
- capture an important decision made in chat

### Knowledge

Examples:

- add a reference file to `knowledge/`
- suggest attaching a source document
- suggest linking an important knowledge file from memory

### Behavior

Examples:

- refine `SOUL.md`
- adjust tone or output structure
- enable a domain-specific assistant profile

## Suggestion Statuses

- `pending`
- `accepted`
- `dismissed`
- `deferred`

## Initial UI Requirements

The first version should include:

- a list of suggestions
- category label
- reason/explanation
- target file label
- accept action
- dismiss action
- defer action

The first version does not need:

- file diffing
- automated file writes
- agent execution
- background scheduling

## v1 Success Criteria

A user can:

- see suggested brain improvements
- understand why each suggestion exists
- accept or dismiss suggestions
- feel that the app is beginning to personalize around them

## v1 Data Shape

Each suggestion needs:

- `id`
- `title`
- `summary`
- `kind`
- `target`
- `reason`
- `status`
- `createdAt`
- optional `proposedContent`
- optional `questionPrompt`

## Future Integration

Later, suggestions can be generated by:

- onboarding answers
- repeated chat themes
- daily log analysis
- user-supplied documents
- cowork/curator agent outputs

The suggestion system should become the approval layer between AI-generated proposals and the user's actual memory files.
